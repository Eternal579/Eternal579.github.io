Servlet工作原理

狭义的Servlet指java语言实现的一个接口，广义的Servlet指任何实现了这个Servlet接口的类。

  1. web服务器（Servlet容器）接收到客户端请求，容器创建“请求和响应”对象，并判断请求的Servlet对象是否存在
  2. 如果存在，则直接调用此Servlet对象的Service方法（间接调用doPost方法和doGet方法），并将“请求和响应”对象作为参数传递
  3. 如果不存在，容器负责加载Servlet类，创建Servlet对象并实例化，然后调用Servlet的init方法进行初始化，之后调用Servlet方法。
  4. 在Service方法中，通过请求对象获取客户端提交的数据并处理，然后通过响应对象将处理结果返回客户端
  5. Servlet生存期由javax.servlet.servlet接口的init,service和destory方法表达

HttpServlet类（重点介绍）

它拓展了GenericServlet类，增加了一新的service（）方法。继承HttpServlet类可以简化Servlet的开发过程，可以通过重写其提供的方法来处理不同类型的HTTP请求。（所以继承HttpServlet就只需要重写doGet()，doPost()方法等）
GET 请求来自于一个 URL 的正常请求，或者来自于一个未指定 METHOD 的 HTML 表单，它由 doGet() 方法处理。

POST 请求来自于一个特别指定了 METHOD 为 POST 的 HTML 表单，它由 doPost() 方法处理。

doGet和doPost的区别

  • doGet 方法：当客户端发送 HTTP GET 请求时，服务器会调用 doGet 方法来处理该请求。通常，GET 请求用于获取资源或者从服务器获取数据，例如通过 URL 参数传递数据。
  • doPost 方法：当客户端发送 HTTP POST 请求时，服务器会调用 doPost 方法来处理该请求。POST 请求通常用于向服务器提交数据，例如通过表单提交数据或者发送 JSON 数据。
