  2. strncpy函数注意点
    char *strncpy(char *dest, const char *src, size_t n);
    如果src的长度大于n时，dest并不会添加’\0’，所以一定要注意！！！

指针

  1. C语言中，指针作为参数传递给函数时，是值传递，函数接收到的是指针的副本，但指向的是同一块内存。
    所以，如果原始指针origin,函数参数叫copy,在函数中产生了一个新的指针叫new，想要通过copy来把new所指的对象赋值给origin所指对象，绝对不能使用copy = new！！！ 一定要使用*copy = *new！！！
  2. memcpy中，如果des和src类型不同，一定要在前面套一个(void *)，这样子在des和src通过加数来进行指针移动操作就不可能出错，不然有的时候会出错; 同时需要记住，对于自定义结构的数组，一定要使用&arr[index]的形式表示偏移量，而不要使用+index的形式，这样子在偏移量大的时候，极其容易偏移出错。
  3. 举个例子 在以上结构中，切记切记，不要尝试short int *block_no_2 = (short int *)(secondary_dblock + k * sizeof(short int));的写法！！！因为这样子很有可能在类型强转的时候出错，而且指针移动的字节数也不一定是k * sizeof(short int)，一定要写成short int *block_no_2 = (short int *)(&secondary_dblock->data[k * sizeof(short int)]);

一些好习惯

  1. 在填写一些size_t类型的函数参数时，一定要养成习惯，能写sizeof(struct DataType)的一定不要写sizeof(variable)！！！
