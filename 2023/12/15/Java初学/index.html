Java初学

一些与C++不同的地方

char是2个字节

byte，short和char在一起运算时，首先转换为int类型进行运算

Java对“+”运算进行了拓展，可作字符串连接运算符，如

>>与>>>

>>表示右移，左边补充符号位
>>>表示无符号右移，左边补0

Java中变量后面的方括号内不能指定数组的大小

使用new为二维数组分配空间

使用break跳转到标号语句

在java中，equals()方法用于比较两个字符串对象的内容是否相等，而==运算符用于比较两个对象的引用是否相等，即只判断是否指向同一个内存地址

java中new返回的是对象的引用，而C++返回的是对象的指针

Java的类知识

类声明

  • abstract： abstract修饰的类不能被实例化，它含有未实现的方法
  • final：final修饰的类不能被继承
  • extends：extends后面只能跟一个父类名
  • implements：用来表明这个类实现了哪些接口
  • 一个类的main方法必须要用static修饰，因为Java运行时系统在开始执行一个程序前，不会生成类的一个实例

成员变量说明

  • static修饰的成员变量称为类变量或静态变量，不加static修饰的称为实例变量。实例变量依附于具体的对象实例，它的值会因为具体对象实例的不同而不同，而类变量为该类的所有对象共享，它的值不因类的对象不同而不同
  • 切记类方法里不能使用实例变量！！！
  • transient用来声明一个暂时性的变量，用于指示JVM，该变量并不属于对象的永久状态，用于实现不同对象的串行化功能
  • volatile声明一个多线程共享变量，使得各个线程对该变量的访问能保持一致
  • synchronized修饰的方法执行之前给方法设置同步机制，实现线程同步
  • naive修饰的方法为本地方法，即方法实现与本地计算机系统有关
  • 方法名一般约定，第一个字母小写，其他单词的第一个字母大写

Java接口说明

  • 定义接口使用interface
  • 接口中的方法只有方法签名，没有方法体。接口中的方法默认为抽象方法，不需要使用abstract关键字进行修饰
  • 一个类可以通过implements关键字实现一个或多个接口。实现接口的类必须提供接口中定义的所有方法的具体实现
  • 通过extends关键字来继承其他接口

访问权限

default说明没有修饰的访问权限限定在同一类和包中

    则会使这个成员变量变成了常量
  • final修饰方法

this

  • 在一个类中创建另一个类的对象时使用
  • 形参与成员变量名称相同

instanceof

一个双目运算符，当左边操作元是右边创建的对象时，返回true

接口

java不允许多继承，从而可以避免菱形继承问题中的函数名相同的问题。java使用接口来替代多继承，因为接口只定义函数，不实现函数，所以不会出现函数同名导致编译器不知道调用哪个函数。接口可以继承多个父接口！！！类中的变量还是方法都默认为default，而接口中的方法默认为public。

枚举类型

枚举常量全部被隐式声明为公有、静态成员，且类型就是声明的枚举类型。

上转型对象

对象的上转型对象的实体是子类负责创建的,但上转型对象会失去原对象的一些属性和功能。上转型对象具有以下特点：

  • 上转型对象不能操作子类新增的成员变量和子类新增的方法。
  • 上转型对象既可以操作子类继承或重写的成员变量,也可以使用子类继承的或重写的方法。
  • 如果子类重写了父类的某个方法后,当对象的上转对象调用这个方法时一定是调用了这个重写的方法,因为程序在运行时知道,这个上转对象的实体是子类创建的,只不过损失了一此功能而已。
  • 可以将对象的上转型对象再强制转换到一个子类对象，这时该子类对象又具备了子类所给的所有属性和功能。

StringTokenizer类

三个构造方法：

  1. StringTokenizer(String str)
  2. StringTokenizer(String str, String delim)
    这个构造方法接受一个字符串 str 和一个分隔符字符串 delim 作为参数，将字符串 str 按照指定的分隔符 delim 进行分割。
  3. StringTokenizer(String str, String delim, boolean returnTokens)
    这个构造方法接受一个字符串 str、一个分隔符字符串 delim，以及一个布尔值 returnTokens 作为参数。returnTokens 决定是否将分隔符作为标记返回。

Java中的异常处理

异常类的类层次

处理机制

  1. 捕获异常
  • 获异常的顺序和catch语句的顺序有关，当捕获到一个异常时，剩下的catch语句就不再进行匹配。因此，在安排catch语句的顺序时，首先应该捕获最特殊的异常，然后再逐渐一般化。也就是一般先安排子类，再安排父类。
  • 捕获异常的最后一步是通过finally语句为异常处理提供一个统一的出口，使得在控制流转到程序的其它部分以前，能够对程序的状态作统一的管理。一般是用来关闭文件或释放其他的系统资源。
  • try-catch-finally可以嵌套
  1. 声明抛弃异常
  2. 抛出异常
  • 抛出异常就是产生异常对象的过程
  • 可以抛出的异常必须是Throwable或其子类的实例
  • 自定义异常类必须是Throwable的直接或间接子类
