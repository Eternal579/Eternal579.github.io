JSP基础

JSP语法

首先jsp最常用的就是用<% your jsp code %>语法。

JSP声明语法

声明变量

  • 这些变量在整个JSP页面内有效。
  • 当用多个用户请求一个JSP页面时，JSP容器为每个客户起动一个线程，这些线程共享JSP页面的成员变量，因此任何一个用户对JSP页面成员变量的操作的结果，都会影响到其他用户。

声明方法

  • 需要注意的是，在方法中定义的变量，只在该方法内有效。

声明类

  • 由于该类在Servlet类的内部，以 内部类 形式存在，所以该类只在本JSP页面内有效，即在JSP页面的Java程序部分可以使用该类创建的对象。

JSP表达式语法

表达式是由常量、变量和运算符组成的式子，在请求处理的时候表达式被计算并插入到输出流返回给客户端。
<%=三个字符之间不能有空格，表达式之后不能有;。

JSP注释语法

<%-- your comment --%>

JSP生命周期

编译阶段：
servlet容器编译servlet源文件，生成servlet类
当浏览器请求JSP页面时，JSP引擎会首先去检查是否需要编译这个文件。如果这个文件没有被编译过，或者在上次编译后被更改过，则编译这个JSP文件。
编译的过程包括三个步骤：

  • 解析JSP文件。
  • 将JSP文件转为servlet。
  • 编译servlet。

组成

JSP页面内容可以分成 元素 和 模板数据 两部分。

  • 元素是在JSP基本语法中定义的内容
    JSP容器在转换阶段将元素翻译成相应的Java代码。
    JSP定义的元素有四种类型：指令、脚本、动作和表达式语言。
  • JSP页面中其他的所有内容都是模板数据。
    JSP容器对模板数据不做处理，例如HTML内容，会直接送到客户端执行。

JSP指令

JSP（JavaServer Pages）指令是用于指导 JSP 容器在编译和执行 JSP 页面时的行为的特殊指令。它们以<%@开始，以%>结束，并位于 JSP 页面的顶部。

  • page 指令：page 指令用于设置整个 JSP 页面的属性和配置选项。它可以指定页面的编码方式、脚本语言、缓冲区大小等
  • include 指令：include 指令用于在 JSP 页面中包含其他文件。它可以将其他 JSP 页面、HTML 文件或文本文件包含到当前页面中
  • taglib 指令：taglib 指令用于引入自定义标签库（Tag Library）到当前的 JSP 页面中。它指定了标签库的 URI（Uniform Resource Identifier）和位置。

JSP隐含对象

也被称为预定义变量

  • request：表示客户端发起的 HTTP 请求的请求对象。它提供了对请求参数、请求头、请求方法等的访问，以及与请求相关的操作和处理。
  • response：表示服务器对客户端的 HTTP 响应的响应对象。它提供了向客户端发送响应内容、设置响应头、重定向等操作的方法。
  • out：表示向客户端发送响应的输出流对象。它提供了向客户端输出文本、HTML、XML 等内容的方法。
  • session：表示客户端与服务器之间的会话对象。它提供了在会话期间存储和检索数据的方法，以及与会话相关的操作，如会话失效、会话跟踪等。

JSP动作

JSP（JavaServer Pages）动作是一种特殊的标记，用于在 JSP 页面中执行特定的操作。这些动作以 jsp: 开头，通过使用不同的动作标签，可以实现各种功能和操作，如控制流程、访问数据库、处理表单数据等。

  • <jsp:include>：用于包含其他页面或文件的内容。可以将一个页面的内容嵌入到另一个页面中，实现代码的重用和模块化。 属性： 描述
    page：包含在页面中的相对URL地址。
    flush：布尔属性，定义在包含资源前是否刷新缓存区。
  • <jsp:forward>：用于将请求转发到另一个页面或资源。可以实现页面之间的跳转，将请求传递给其他页面进行处理。
  • <jsp:useBean>：装载一个将在JSP页面中使用的 JavaBean
