HTTP

HTTP默认端口号为80

HTTP工作原理

  • HTTP是无连接的：指限制每次连接只处理一个请求。服务器处理完客户的请求，并收到客户的应答后，即断开连接。采用这种方式可以节省传输时间
  • HTTP是媒体独立的：指任何类型的数据，客户端和服务器都可以通过HTTP发送
  • HTTP是无状态的：指协议对于事务处理没有记忆能力

HTTP消息结构

客户端请求

  1. 请求行
  2. 请求头部
  3. 空行
  4. 请求数据

服务器响应消息

  1. 状态行
  2. 消息报头
  3. 空行
  4. 响应正文

8种请求类型

只介绍两种

  1. GET：向特定的资源发出请求
  2. POST：向指定资源提交数据进行处理请求
    URL中，”?”后面的内容为参数信息，多个参数之间使用”&”分隔
    POST比GET请求方式更安全

HTTP状态码

第一个十进制数字定义了状态码的类型

  • 200 请求成功
  • 301 资源被永久转移到其它URL
  • 404 请求的资源不存在
  • 500 内部服务器错误
